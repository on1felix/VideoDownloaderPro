# Исправления загрузки yt-dlp и FFmpeg

## Проблема
На некоторых ПК возникали ошибки при скачивании yt-dlp и FFmpeg из-за:
- Отсутствия таймаутов соединения
- Проблем с SSL сертификатами
- Отсутствия механизма повторных попыток
- Недоступности основного источника (GitHub)

## Как работает система зеркал

**Алгоритм:**
1. Пробует основной URL (GitHub latest)
2. Если не получается - пробует 2 раза с задержкой 2 сек
3. Если все попытки неудачны - переключается на зеркало #1
4. Повторяет попытки для зеркала #1
5. Если не получается - переключается на зеркало #2
6. И так далее, пока не скачает или не закончатся зеркала

**Пример логов:**
```
[INFO] Downloading yt-dlp: 10.5 MB
[WARN] Download failed: timeout. Retrying...
[WARN] Mirror failed: timeout. Trying next mirror...
[INFO] Trying alternative mirror #1...
[SUCCESS] Download completed!
```

## Технические детали

### SSL контекст
```python
ssl_context = ssl.create_default_context()
ssl_context.check_hostname = False
ssl_context.verify_mode = ssl.CERT_NONE
```
Отключает проверку SSL сертификатов для обхода проблем на корпоративных сетях.

### Retry логика
- 2 попытки для каждого зеркала
- Задержка 2 секунды между попытками
- Логирование каждой попытки и переключения зеркал

### Таймауты
- 30 секунд на установку соединения
- Предотвращает зависание при проблемах с сетью

### Зеркала
- **yt-dlp**: 3 зеркала (GitHub latest, конкретная версия, CDN)
- **FFmpeg**: 2 зеркала (yt-dlp/FFmpeg-Builds, BtbN/FFmpeg-Builds)
